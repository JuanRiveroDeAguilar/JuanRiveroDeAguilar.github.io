Capítulo 2. Hola mundo.

El famoso “Hola mundo” en programación, ¡pero a mí me costó un mundo ejecutar este script!.

print("Hola mundo")

¿Pero qué es "Hola mundo"? ¿Como se ejecuta?.

A modo de resumen habría que:

  1. Escribir un código en un editor de texto.
  2. Guardar el archivho con una extensión adecuada (.py para Python).
  3. Ejecutar en la terminal.

Esto que parece fácil no es tan intuitivo para el que se acaba de inciar en la programación, ya que hay que instalar lenguajes y preparar el entorno para ejecutar el código. En mi caso, al principio instalé Python y ejecuté el archivo desde allí, pero después no sabía si era mejor importar el archivo .py, ejecutarlo con un doble click o abrirlo en el IDLE (el entorno de desarrollo integrado de Python). En resumen, es muy fácil perderse en el proceso.

Pasos a seguir para ejecutar un código en Python:

  1. Primero necesitamos un editor de texto adecuado para escribir código como es Visual Studio Code. Facilita mucho la escritura y lectura, además permite abrir la terminal.
  2. A continuación, creamos un archivo de texto desde VSC y lo guardamos como hola.py.
  3. Ahora escribimos el código: 
    print("Hola mundo")
  4. Finalmente, ejecutamos el script abriendo desde la terminal dentro de VSC e indicando la versión de Python que estamos utilizando (en este caso la 3.12): 
    python3.12 hola.py

    5. Si todo va bien debería imprimirse:

    (base) MacBook-Air-de-Juan:python juan$ python3.12 hola.py
Hola mundo

Aquí nos detenemos un momento, ya que es necesario explicar un par de puntos importantes. Para poder ejecutar hola.py, tenemos que indicar a Python el directorio en donde se encuentra el archivo (o bien situarnos en ese directorio desde la terminal). Una forma sencilla de averiguarlo es escribir en la terminal: pwd

(base) MacBook-Air-de-Juan:~ juan$ pwd
/Users/juan

En este caso, el directorio está en /Users/juan. En el supuesto de que no sea el correcto, habría que desplazarse mediante los típicos comandos de UNIX de la terminal (cd, cd.., ls) hasta llegar al archivo.

Por ejemplo, si el archivo hola.py está en /Users/juan/Desktop/python/hola.py hay que escribir en la terminal cd desktop, a continuación cd python, y finalmente ejecutar el programa python3.12 hola.py (también se podría ejecutar indicando toda la ruta python3.12 /Users/juan/Desktop/python/hola.py). 

(base) MacBook-Air-de-Juan:~ juan$ cd desktop
(base) MacBook-Air-de-Juan:desktop juan$ cd python
(base) MacBook-Air-de-Juan:python juan$ python3.12 hola.py
Hola mundo

Bien, acabamos de ejecutar nuestro script "Hola mundo", ¡Enhorabuena!.

¿Pero qué sucedería si en vez de escribir "Hola mundo" en hola.py escribimos "ATCG"?

(base) MacBook-Air-de-Juan:python juan$ python3.12 hola.py
ATGC

Capítulo 3. Trabajando con cadenas (strings).

Para entender bien que son las cadenas crearemos un archivo dna.py con el fin de imprimir la secuencia AAA en la terminal. Aquí ya estamos entrando en programación Python y hay una serie de reglas (o sintaxis) que hay que seguir escrupulosamente para que se ejecute el script correctamente.

Por ejemplo, si en nuestro script dna.py escribimos:

print(AAA)

y ejecutamos en la terminal, nos da el siguiente error:

(base) MacBook-Air-de-Juan:python juan$ python3.12 dna.py
Traceback (most recent call last):
    File "/Users/juan/Desktop/python/dna.py", line 1, in <module>
    print(AAA)
        ^^^
NameError: name 'AAA' is not defined

Este error nos indica de que no hemos definido todavía el tipo de variable. En nuestro caso, como es una secuencia de caracteres alfabéticos equivale a una cadena (string), por lo que tenemos que indicarlo con comillas simples (o dobles). Si nos fijamos más arriba, "Hola mundo" lo habíamos escrito entre comillas.

Entonces si ahora escribimos print("AAA") y ejecutamos en la terminal, se imprimirá correctamente:

(base) MacBook-Air-de-Juan:python juan$ python3.12 dna.py
AAA

En resumen, es así como analizamos los datos genéticos. Partimos de unos caracteres alfabéticos (secuencias) y mediante un lenguaje de programación realizamos los análisis.

En este momento, podríamos trabajar perfectamente con cadenas. Por ejemplo, si creaamos una variable secuencia en dna.py (en vez de ejecutar directamente print("AAA")) y ejecutamos:

secuencia = "AAA"
print(secuencia)

resulta en:

(base) MacBook-Air-de-Juan:python juan$ python3.12 dna.py
AAA

A su vez, podríamos intentar convertir la secuencia en minúsculas (como en cualquier editor de texto) con la función variable.lower(), simplemente añadiendo al código:

secuencia = "AAA"
print(secuencia)
secuencia = secuencia.lower()
print(secuencia)

dando como resultando:

(base) MacBook-Air-de-Juan:python juan$ python3.12 dna.py
AAA
aaa

Es decir, obtendríamos tanto el print de las mayúsculas AAA como de las minúsculas aaa.

Pero aquí otro punto importante. Aunque podamos trabajar directamente con cadenas, en bioinformática por lo general trabajamos con texto que tiene significado biológico. Es decir, ya no trataríamos a una A como un letra, sino como al nuceótido A (Adenina). En Python, podemos trabajar con librerías específicas que nos van a permitir realizar análisis de secuencias, como Biopython, el cual trata a los datos como datos biológicos.

Capítulo 4. Biopython.

Biopython es un proyecto con múltiples herramientas para analizar datos genéticos en forma de secuencias biológicas. Estas secuencias se pueden obtener tanto de investigaciones propias como de bases de datos tipo GenBank.

Trabajando con Biopython:

Una vez instalado, lo que debemos hacer es llamar a los módulos que queremos utilizar, las cuales contendrán las funciones de interés. En nuestro ejemplo, vamos a tratar de convertir nuestra secuencia AAA en su cadena complementaria. En nuestro código, primero importamos la clase Seq desde el módulo Bio.Seq. Seq sirve para convertir nuestra cadena en un objeto de clase Seq (con sentido biológico) y a continuación llamamos a la función que queremos utilizar, concretamente seq.complement():

from Bio.Seq import Seq
secuencia = Seq("AAA")
print(secuencia)
print(secuencia.complement())

y obtendríamos:

(base) MacBook-Air-de-Juan:python juan$ python3.12 dna.py
AAA
TTT

Si por ejemplo, quisiésemos además obtener la secuencia complementaria inversa simplemente habría que utilizar la función reverse_complement():

from Bio.Seq import Seq
secuencia = Seq("AAA")
print(secuencia)
print(secuencia.complement())
print(secuencia.reverse_complement())

y obtendríamos:

(base) MacBook-Air-de-Juan:python juan$ python3.12 dna.py
AAA
TTT
TTT

En este caso la cadena complementaria inversa es igual a la complementaria, pero se debe a que estamos partiendo de una secuencia muy corta y con los mismos nucleótidos AAA, pero si en vez de AAA partiésemos con una secuencia más larga y con mayor variedad de nucleótidos, por ejemplo AAATTTCCC obtendríamos: 

(base) MacBook-Air-de-Juan:python juan$ python3.12 dna.py
AAATTTCCC
TTTAAAGGG
GGGAAATTT

Aquí ya podemos observar que la complejidad de información va a depender de la longitud de la secuencia, que puede ser de pocas pares de bases hasta alcanzar el nivel de genoma. Para hacernos una idea, el ADN humano está formado por una secuencia lineal de aproximadamente 3.272.116.950 nucleótidos, ordenados uno detrás de otro como letras en un texto, lo que equivaldría a varios gigabytes de información.

Es posible realizar análísis hasta cierta complejidad en nuestro propio ordenador, sin embargo cuando manejemos grandes cantidad de datos dependemos completamente de supercomputadores y/o servicios de computación en la nube.
